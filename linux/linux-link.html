【Linux】软连接与硬链接

发表于 2018-03-05 | 分类于 linux |

Linux中的文件

Unix总除进程之外，一切皆为文件。但Linux在文件系统中引入了目录的概念，方便文件的管理，使得文件可被分类存放。目录的引入使得Linux文件系统形成一个层次结构的目录树，而目录文件本身也是一个文件。

元数据与用户数据

在Linux中，文件分为元数据(metadata)与用户数据(user data)。用户数据，即文件数据块(data block)，数据块是记录文件真实内容的地方；而元数据是文件的其他属性的描述，如文件名、创建时间、所有者等。元数据中的inode号(索引节点号)，才是文件的唯一标识而非文件名。文件名仅仅是为了方便使用者记忆和使用的，系统和程序通过inode号找到正确的文件数据块。

比如, 通过文件名打开一个文件:
[filename->inode]->(data blocks)
[]表示metadata，()表示user data

什么是硬链接

Linux系统中引入了两种链接: 硬链接(hard link)与软链接(soft link/symbolic lik)。若一个inode号对应多个问及爱你名，则这些文件之间为硬链接。

硬链接的存在以下几个特点:

  1. 文件有相同的inode及datablock
  2. 只能对已存在的文件进行创建
  3. 不能交叉文件系统进行硬链接的创建(不是一颗索引)
  4. 不对能对目录进行创建
    受限于文件系统的设计，例如.与..其实就是硬链接，如果系统允许用户自己创建硬链接，将产生目录循环。
  5. 删除一个硬链接文件并不影响其他有相同inode号的文件
    当inode存在切链接计数器(link count)不为0时，inode的所有者才是唯一的。

需要注意的是，当inode号耗尽时，尽管还有剩余空间，也不能再创建文件！

什么是软链接

软连接与硬链接不同，若一个文件的用户数据块存放的是另一个文件的路径名指向，则该文件就是软链接文件。软链接文件就是一个普通文件，只是数据块内容比较特殊，软链接文件有自己的inode号以及用户数据块，因此它的创建并不存在那么多的限制。

  • 软链接有自己的文件属性及权限
  • 可对不存在的文件或目录创建软链接
  • 软链接可交叉文件系统
  • 可对文件和目录创建
  • 创建软链接不会导致inode的link count计数器增加
  • 删除软链接不会影响源文件，但若源文件被删除，则该软连接被称之为死链接(dangline link)，若被指向路径文件被重新创建，死链接又恢复为正常的软链接。

使用软链接访问一个文件:
soft-link -> inode -> user data -> src-file -> inode -> user data;

同时，软链接也可以是另一个软链接的路径，其解析过程是递归的。

需要注意的是，软链接创建时最好使用绝对路径，使用相对路径会使得软链接文件被挪动后导致其成为一个死链接。
